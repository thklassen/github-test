C# Instrumentation Sample of an Application and a DLL

Setup

Use "Start→Squish Coco→Build Environment Selection" to select the Visual Studio toolchain to use.

Compiling

"Step 1 Build With Command Line Compiler" build:
  • hellodll.dll and hellodll.dll.csmes which contains the instrumented code of hello.cs
  • hello.exe and hello.exe.csmes which contains the instrumented code of hello.cs (included automatically when linking again hellodll.dll) and tutorial.cs

Executing

Manual Execution

"Step 2 Execute Application" opens a console window which permits to execute hello.exe. After each execution of it, a file hello.exe.csexe is generated which is the execution report.

Batch Execution

"Step 2 Execute Test Suite" is a small batch file which execute several times hello.exe. This batch file also set the execution name and the status (passed/fail) of each run.

Coverage Analysis

Main Application and DLL

"Step 3 Code Coverage Analysis of hello.exe" opens CoverageBrowser and permits to analyze the code coverage of the code in the DLL (hellodll.dll) and the main application (hello.exe).

DLL Only

"Step 3 Code Coverage Analysis of hellodll.dll only" opens CoverageBrowser and permits to analyze the code coverage of the code in the DLL (hellodll.dll) only.
